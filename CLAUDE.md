- Repo Stack

  - Package Manager
    - Bun
  - Monorepo architecture. Check apps directory
  - The deployments happen on railway using the Dockerfile in the root of the repo.

- Things to remember before starting a task
  - Familiarize yourself with relevant code sections and documentation
  - Commit related changes together, with clear commit messages.
  - Divide large tasks into smaller, manageable units each with their separate commits
  - Testing will be handled separately after the code is deployed, so there's no need to include testing steps as part of the task TODO list.
